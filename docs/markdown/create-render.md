# Создание рендера
## Content.MapRenderer - встроеный инструмент который позволяет рендерить карты

</div>

## Настройка рендера

Нужно склонировать репозиторий сервера

Не забудьте перейти в его диреторию

Потом нужно забилдить рендер командой `dotnet build Content.MapRenderer --configuration Release`

## Запуск рендера

Можно запустить рендер командой `dotnet run --project Content.MapRenderer`. Вам предоставится список карт имеющие прототипы далее водете id карты или можете выбрать "all" для рендера всех карт

Вы так же можете сразу запустить рендер своей карты введя `dotnet run --project Content.MapRenderer [id карты]` учтите что для этого нужен прототип самой карты

Рендер так же подерживает выбор нескольких карт (через пробел) для рендера например
`Content.MapRenderer Box Bagel`

## Параметры для рендера

* `--format <png|webp>`
    * Формат в котором будет сохранена картинка карты.
    * По умолчанию: png

* `--viewer`
    * Создаёт map.json карты
    * Файл нужен для MapViwer (сайт с рендерами)

* `--output <output path>`
    * Изменяет путь где будет сохранена картинка рендера.
    * По умочанию: Resources/MapImages

* `--files`
    * Позволяет указать путь карты которую нужно отрендерить
    * Если карта указана путём то прототип для её **НЕ** нужен
    * Пример: Content.MapRenderer --files /Maps/box.yml /Maps/bagel.yml

* `--markers`
    * Показывает маркеры на рендере. По умолчанию: false

* `--parallax`
    * Output images and data used for map viewer parallax.

* `--help`
    * Показывает список параметров
